Suggest Edits

Additionally to the Analytics Options, you can set specific options to a video. The options below can also be defined in the Analytics Options object but will be overwritten if a Video Options object is set after defining the Analytics Options.

More context for the differences between Analytics Options and Video Options is available here.

To set video options you can follow this example:

// Assuming you have already initialized the NPAW Plugin and the Exoplayer instance

// Create the default VideoOptions object and set some options
VideoOptions videoOptions = new VideoOptions.Builder().build();
videoOptions.setContentTitle("NewTitle");
videoOptions.setContentCustomDimension1("param1");
videoOptions.setContentCustomDimension2("param2");
videoOptions.setContentCustomDimension3("param3");
videoOptions.setContentCustomDimension4("param4");
videoOptions.setContentCustomDimension5("param5");

// Set video options for that specfic video player
VideoAdapter videoAdapter = npawPlugin.videoBuilder()
    .setPlayerAdapter(new ExoPlayerAdapter(this, player))
    .setOptions(videoOptions)
    .build();

// Assuming you have already initialized the NPAW Plugin and the Exoplayer instance

// Create the default VideoOptions object and set some options
val videoOptions = VideoOptions.Builder().build()
videoOptions.contentTitle = "NewTitle"
videoOptions.contentCustomDimension1 = "param1"
videoOptions.contentCustomDimension2 = "param2"
videoOptions.contentCustomDimension3 = "param3"
videoOptions.contentCustomDimension4 = "param4"
videoOptions.contentCustomDimension5 = "param5"

// Set video options for that specfic video player
val videoAdapter = npawPlugin.videoBuilder()
    .setPlayerAdapter(ExoPlayerAdapter(this, player))
    .setOptions(videoOptions)
    .build()

Parameter Description

ParametersTypeDescription
contentTitlestringYou must set this parameter to let users search and filter real time data.
programstringIf you set this parameter, what you set will be displayed as Program on NPAW Suite. This can be changed multiple times during the view (e.g., user is watching a single TV channel, but the programs/TV shows keep changing).
contentDurationintegerMedia duration in seconds: This parameter is drawn from the player, but will be overwritten should the plugin be integrated.
livebooleanThis parameter indicates whether a stream URL resource is a live stream or Video on Demand (VoD) content. Accordingly, if true, then our Analytics app will treat that content as a Live stream. If false, then it will treat that content as VoD.
contentFpsnumberFrames per second of the content in the current moment.
refererstringSet the URL of the site. In the NPAW Suite this is shown as "Content > Domain" dimension. Only the second -and top- level domain is stored, the rest of the URL is discarded. E.g., reporting "http://www.npaw.com/about/index.html" will result in "npaw.com" shown in the "Content > Domain" dimension.
contentBitrateintegerBitrate of the content being reproduced.
contentTotalBytesintegerTotal downloaded bytes of the content.
contentThroughputnumberThroughput of the client bandwith in bits per second.
contentSendTotalBytesbooleanReport totalbytes or bitrate, default false (bitrate).
contentRenditionstringTag of the video quality being reproduced.
contentResourcestringMedia URL: This must be set when integrating the plugin.
contentCdnstringThe original CDN is automatically detected from the media resource host. But it can be overwritten with this parameter, if needed. Find the CDNs list here. You need to use the “code” string.
contentCdnNodestringCDN node id.
contentCdnTypestringCDN node content access type. It defines if the content request hits the cache or not - TCP_HIT / HIT: 1,TCP_MISS / MISS: 2,TCP_MEM_HIT:3,TCP_IMS_HIT:4
contentStreamingProtocolstringUsed to manually set the content's media streaming protocol. Options include: HDS (Adobe HDS), HLS (Apple HLS), MSS (Microsoft Smooth Streaming), DASH (MPEG-DASH), RTMP (Adobe RTMP), RTP, RTSP, MULTICAST, DVB, DVB-C, DVB-T, DVB-T2.
contentMetadataobjectObject containing mixed extra information about the content like director, parental rating, device info or the audio channels.This object may store any serializable key:value info.
contentPackagestringFilter your data by a specific package such as Warner Bros, Paramount, Disney, Netflix, HBO, etc.
contentSagastringFilter your data by a specific saga, known as a collection of related, successive films that share the same fictional universe, or are marketed as a series. Note, saga dimensions for VOD movies are equivalent to VOD TV shows.
contentTvShowstringFilter your data by the TV show that is being consumed. If the video content is not identified as a TV show, it will not be included in the dataset.
contentSeasonstringFilter your data by a specific season or sequence of related series programs, such as first, second, or third season of a specific series.
contentEpisodeTitlestringFilter your data by the episode title. This field should be used for content that has this relevant field (eg. series instead of movies).
contentChannelstringFilter your data by the channel being tuned into to view content.
contentIdstringFilter your data by a specific content ID (internal content identifier code specifically set by each customer).
contentImdbIdstringFilter your data by a specific IMBD ID.
contentGracenoteIdstringFilter your data by a specific Gracenote ID.
contentTypestringFilter your data by a specific content type such as trailer, main, extra, clip, preview.
contentGenrestringFilter your data by a specific content genre such as romance, thriller, fantasy, among many others.
contentLanguagestringFilter your data by the language a user sets when watching content. NPAW Suite may make a record of multiple values during the video view. This can be done by changing the value in the Options object. Please note that multiple values are displayed in NPAW Suite in the Single Video View Details screen of the Tracking Video section. For aggregated dashboards and in the filter picker, only the last value recorded before the JOINTIME event is taken into account.
contentSubtitlesstringFilter your data by the subtitle a user sets when watching content. NPAW Suite may make a record of multiple values during the video view. This can be done by changing the value in the Options object. Please note that multiple values are displayed in NPAW Suite in the Single Video View Details screen of the Tracking Video section. For aggregated dashboards and in the filter picker, only the last value recorded before the JOINTIME event is taken into account.
contentContractedResolutionstringFilter your data by a specific package quality contracted, also known as commercialized resolution, with the objective to distinguish the consumption of customers with different resolution contracts (e.g., platform plans with access to SD, HD or UHD qualities).
contentCostdoubleFilter your data by the cost of the subscription that the user has signed up for.
contentPricedoubleFilter your data by the price which the customer has paid to view a specific piece of content. This is only applicable when a user has made a transaction for this purchase.
contentPlaybackTypestringFilter your data by the type of media, whether this is LIVE, VOD or both.
contentDrmstringFilter your data by the Digital Rights Management technology being used to encrypt and protect ad content.
contentEncodingVideoCodecstringFilter your data by the video codec used within the container format.
contentEncodingAudioCodecstringFilter your data by the audio codec used within the container format.
contentEncodingCodecSettingsobjectFilter by the different encoding settings the customer has set in the encoder.
contentEncodingCodecProfilestringCodec profiles target specific classes of application. Filter your views by their codec profile to see content with specific requirements.
contentEncodingContainerstringFilter your data by the container format of the content file.
contentMetricsMap<String, Any>[Map] containing the metric name as String and the numeric value as the value. It accepts float, double, int and long as values, any other type will get ignored.
contentTransactionCodestringThis is a specific capable to search for, or identify and view, all users available within our Analytics platform. This code can be configured in several ways: purchaseCode, sessionCode, etc. Typically, you should use a parameter the customer can relate to their content management system (CMS) or paid transactions.

VideoOptions videoOptions = new VideoOptions.Builder().build();

Parameter Description

ParametersTypeDescription
networkIPstringThe original IP is automatically detected based on the X-Forwarded-For HTTP header, or based on the IP address of the HTTP request. Use this parameter to overwrite it, if needed. Any desired value can be set.
networkIspstringThe original ISP is automatically detected from a third-party library. But use this parameter to overwrite it, if required. Any desired value can be set.
networkConnectionTypestringThe connection type, if needed.

Parameter Description

The table below describes each of these parameters:

ParametersTypeDescription
isParseCdnNodeBooleanIf true the plugin will query the CDN to retrieve the node name. It might slow performance down. Default: false. For more information, visit: here
parseCdnNodeListArrayList<String>List of CDN names to parse. This is only used when isParseCdnNode is true. Order is considered when trying to match against the CDN. Default: ["Akamai", "Cloudfront", "Level3", "Fastly", "Highwinds", "Telefonica", "Amazon", "Edgecast", "NosOtt"]. For more information, visit: here
parseCdnNameHeaderListArrayList<String>If defined, resource parsing will try to fetch the CDN code from the custom headers defined by this property. Default: ["x-cdn-forward"]. For more information, visit: here
parseCdnNodeHeaderListArrayList<String>If defined, resource parsing will try to fetch the CDN node from the custom header defined by this property. For example: ["x-node"]. For more information, visit: here
parseCdnNodeAllowedGetMethodOnRetryBooleanIf true, in case of an unsuccessful HEAD request for CdnNode, it will retry with the GET method. It might slow performance down. Default: true. For more information, visit: here
parseCdnNodeRequestDelayedBooleanIf true, the plugin will only query the CDN to retrieve the node name after the first frame is rendered. Default: false.
customCdnParseListArrayList<String>This is a list of custom CdnParse builders. Each builder in the list provides a way to generate a CdnParse instance with different parsing behaviors based on its own configuration. For more information, visit: here
isParseManifestBooleanIf true, all "content" parsers will be used, HLS or DASH and location. The plugin will parse the resource trying to find out the real given resource instead of the API url. Default: false. For more information, visit: here
parseManifestAuthMap<String, String>If isParseManifest enabled, it adds extra headers to the request of the content. Use this if, for example, the player needs to include authentication headers to request the content, so the plugin needs it to access the manifest files too. For more information, visit: here
transportFormatStringAllows manual specification of the transport stream format used in the manifest. When set, this will override any automatic detection, and it must be one of the following accepted values: 'TS' for MPEG Transport Stream, 'MP4' for MPEG-4, or 'CMF' for the Common Media Application Format. Custom Dimensions

If additional parameters are required while filtering data in our analytics app, optional integrations or custom dimensions are available. There are 20 possible custom dimension set up within the device’s player when integrating the plugin.

Anything which requires filtration in real-time, and is of a relatively small value, is capable of becoming a custom dimension. For example, if a news website contains video in its several “sections” (e.g., sports, family, humor, kids, finance, etc.), a custom dimension titled “section” can be created with the above values. Please note, however, that such dimensions possess a limited number of well-controlled values.

Another possible custom dimension could collect demographic information for logged end users. Dividing an audience by gender and 5 age ranges, such as “male 24-35” or “female 60+”, the result will be a range of 20 possible values. This custom dimension could be titled “demographics”.

Serialization and Deserialization of Video Options

In scenarios where you need to persist or transfer the state of VideoOptions across different parts of your application or between video sessions, the serialization and deserialization feature comes in handy. This feature allows you to convert VideoOptions into a Bundle format and vice versa, enabling easy storage or transmission.

Serialization to Bundle with toBundle()

The toBundle() method converts the current state of an VideoOptions object into a Bundle. This Bundle can be used for purposes such as saving state during configuration changes in Android, passing data between components, or storing settings for later use.

Usage example

// Create and configure an VideoOptions instance
VideoOptions videoOptions = new VideoOptions.Builder().build();
videoOptions.setTitle("Title demo");
videoOptions.setLive(true);

// Serialize VideoOptions to Bundle
Bundle videoOptionsBundle = videoOptions.toBundle();

// Create and configure an VideoOptions instance
val videoOptions: VideoOptions = VideoOptions.Builder().apply { 
    title = "Title demo"
    live = true
}.build()

// Serialize VideoOptions to Bundle
val videoOptionsBundle = videoOptions.toBundle()

Deserialization from Bundle

Deserialization can be performed in two ways: using the Builder pattern or directly setting the Bundle to an existing VideoOptions instance. This flexibility allows you to choose the method that best fits your application's architecture.

Option 1: Import Bundle by Builder

You can use the fromBundle() method in the VideoOptions.Builder to create a new instance of VideoOptions directly from a Bundle. This method is useful when you are starting fresh with a new VideoOptions instance.

Usage Example
// Deserialize VideoOptions from Bundle using Builder
VideoOptions videoOptions = new VideoOptions.Builder()
    .fromBundle(videoOptionsBundle)
    .build();

// Deserialize VideoOptions from Bundle using Builder
val videoOptions = VideoOptions.Builder()
    .fromBundle(videoOptionsBundle)
    .build()

Option 2: Import Bundle Directly to VideoOptions

If you already have an instance of VideoOptions and want to update it with values from a Bundle, you can use the fromBundle() method directly on the instance. This method applies the serialized settings from the Bundle to the existing VideoOptions object.

Usage Example
// Create an empty VideoOptions instance
VideoOptions videoOptions = new VideoOptions.Builder().build();

// Update existing VideoOptions instance from Bundle
videoOptions.fromBundle(videoOptionsBundle);

// Create an empty VideoOptions instance
val videoOptions = VideoOptions.Builder().build()

// Update existing VideoOptions instance from Bundle
videoOptions.fromBundle(videoOptionsBundle)

Parameter Description for Serialization and Deserialization

The table below provides a description of the methods involved in the serialization and deserialization process:

MethodReturn TypeDescription
toBundle()BundleSerializes the current state of the VideoOptions object into a Bundle.
fromBundle()voidUpdates the existing VideoOptions object with values from the provided Bundle. Used directly on an instance.
fromBundle()VideoOptions.BuilderUsed in the Builder to create a new instance that incorporates settings from the provided Bundle.

Practical Applications

  • State Preservation: Useful in Android development for handling configuration changes or preserving state when navigating between activities.
  • Data Transfer: Simplifies the process of passing configuration settings between different parts of an application or even different applications.
  • Persistence: Facilitates saving and retrieving user or application settings locally or remotely, enhancing the user experience by maintaining a consistent state across sessions.

By integrating these serialization and deserialization methods into your application, you can enhance data handling and state management, leading to more robust and user-friendly applications.
